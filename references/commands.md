# Slash Command Reference

## Building a Project

| Command | What it does | When to use it |
|---------|-------------|----------------|
| `/starter-pack` | Sets up your project and starts brainstorming. | Start of any new project. |
| `/office-hours` | Brainstorms your idea. Asks questions to shape what you're building. | When you have an idea and want to think it through. |
| `/plan-ceo-review` | Challenges your scope. Pushes back, finds gaps. | After brainstorming, before building. |
| `/plan-eng-review` | Locks the technical approach. Reviews architecture and edge cases. | After scope is set, before building. |
| `execute-plan` | Builds from the reviewed plan, step by step. | After the plan is locked. |
| `/qa` | Tests the site, finds bugs, takes screenshots, fixes issues. | After building, before shipping. |
| `/ship` | Pushes to GitHub, creates a PR, deploys. | When the project is ready to go live. |

## Managing Your Session

| Command | What it does | When to use it |
|---------|-------------|----------------|
| `/compact` | Summarizes the conversation to free up space. | After finishing a major feature. |
| `/clear` | Resets the conversation. Files and code are untouched. | When switching tasks or when Claude seems confused. |
| `/effort` | Adjusts how hard Claude thinks. High, medium, or low. | High for planning. Medium for building. Low for quick fixes. |

## Useful Things to Say

- "What did you just change?" - Get a summary of recent changes.
- "Undo that." - Revert the last change.
- "Update CLAUDE.md so you don't make that mistake again." - After correcting Claude.
- "Review the codebase for personal information." - Before making a repo public.
- "Push everything to GitHub." - When you're ready to publish.
